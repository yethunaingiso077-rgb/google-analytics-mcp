# How to Contribute

Please read these guidelines thoroughly before contributing. By following these steps, you can help ensure your contributions are as effective and impactful as possible.

## Getting Started
- Fork the repository to your own GitHub account.
- Clone the fork to your local machine.
- Make sure to only work on your fork of the repository.

## Making Changes
- Work on the files you want to change.
- Keep your code clean and adhere to the project's coding style.
- Ensure your changes do not break the existing functionality.
- Test your changes thoroughly.

## Commiting Changes
- Commit your changes with a clear message detailing what you have done.
- Make sure to push your changes to your fork.

## Submitting Changes
- Open a pull request to the main repository.
- Include details about what changes were made and why they are necessary.

## Code Style
- Follow the coding style used in the original files (see original/copyright for more details).
